# Deployment & Staging Setup Guide

This guide details how to deploy the Postgres database, Node.js backend to Google Cloud Run, and the React admin dashboard to Firebase Hosting.

---

## 1. Database Provisioning (Neon Postgres)
Neon provides a fully-managed serverless PostgreSQL database with a free tier.

1. Sign up at [Neon.tech](https://neon.tech/).
2. Create a new project named `goods-delivery-platform`.
3. Choose the region nearest to your customers (e.g. `ap-south-1` for Odisha/India).
4. Copy the **Connection String** from the dashboard. It will look like this:
   `postgresql://neondb_owner:password@ep-cool-snowflake-123456.ap-south-1.aws.neon.tech/neondb?sslmode=require`
5. Save this URL for the backend deployment env variables.

---

## 2. Backend Deployment (Google Cloud Run)
Google Cloud Run hosts containerized microservices and automatically handles scaling.

### Prerequisites
- Install [Google Cloud CLI](https://cloud.google.com/sdk/gcloud).
- Authenticate and set project:
  ```bash
  gcloud auth login
  gcloud config set project [YOUR_PROJECT_ID]
  ```

### Build & Deploy Commands
1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Enable necessary GCP APIs:
   ```bash
   gcloud services enable run.googleapis.com containerregistry.googleapis.com
   ```
3. Submit build to Cloud Build:
   ```bash
   gcloud builds submit --tag gcr.io/[YOUR_PROJECT_ID]/goods-delivery-backend
   ```
4. Deploy the image to Cloud Run:
   ```bash
   gcloud run deploy goods-delivery-backend \
     --image gcr.io/[YOUR_PROJECT_ID]/goods-delivery-backend \
     --platform managed \
     --region us-central1 \
     --allow-unauthenticated \
     --set-env-vars DATABASE_URL="postgresql://neondb_owner:password@ep-cool-snowflake-123456.ap-south-1.aws.neon.tech/neondb?sslmode=require" \
     --set-env-vars PORT=8080
   ```
5. Copy the HTTPS URL generated by Cloud Run (e.g. `https://goods-delivery-backend-xyz.a.run.app`).

## 3. Web Portal Deployment (Firebase Hosting)
Vite generates static assets that can be hosted for free on Firebase Hosting.

### Build and Package
1. Navigate to the web portal directory:
   ```bash
   cd web_portal
   ```
2. Setup production variables in `web_portal/.env`:
   ```env
   VITE_FIREBASE_API_KEY=your-api-key
   VITE_FIREBASE_AUTH_DOMAIN=your-auth-domain
   VITE_FIREBASE_PROJECT_ID=your-project-id
   VITE_FIREBASE_STORAGE_BUCKET=your-storage-bucket
   VITE_FIREBASE_MESSAGING_SENDER_ID=your-sender-id
   VITE_FIREBASE_APP_ID=your-app-id
   ```
3. Generate distribution package:
   ```bash
   npm run build
   ```

### Deploy to Firebase
1. Install firebase-tools globally if not present:
   ```bash
   npm install -g firebase-tools
   ```
2. Initialize hosting:
   ```bash
   firebase login
   firebase init hosting
   ```
   - Select your Firebase project.
   - Set **public directory** to `dist`.
   - Configure as a **single-page app (Rewrite all URLs to /index.html)**: `Yes`.
   - Set up automatic builds with GitHub: `No` (or `Yes` if CI is desired).
3. Deploy the files:
   ```bash
   firebase deploy --only hosting
   ```
4. Firebase will output your live URL (e.g. `https://goods-delivery-platform.web.app`). Add this URL to the Google Cloud Console Authorized Javascript Origins if setting up Google Sign-in.
