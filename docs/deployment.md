# VAYA Deployment & Staging Setup Guide

This guide details how to deploy the Postgres database, Node.js backend to Google Cloud Run, and the React frontends (VAYA Public website and VAYA Operations admin dashboard) to Firebase Hosting.

---

## 1. Database Provisioning (Neon Postgres)
Neon provides a fully-managed serverless PostgreSQL database with a free tier.

1. Sign up at [Neon.tech](https://neon.tech/).
2. Create a new project named `vaya-logistics`.
3. Choose the region nearest to your customers (e.g. `aws-ap-southeast-1` Singapore for Odisha/India operations).
4. Copy the **Connection String** from the dashboard. It will look like this:
   `postgresql://neondb_owner:password@ep-cool-snowflake-123456.ap-southeast-1.aws.neon.tech/neondb?sslmode=require`
5. Save this URL for the backend deployment env variables.

## 2. Backend Deployment (Google Cloud Run)
Google Cloud Run hosts containerized microservices and automatically handles scaling.

### Prerequisites
- Install [Google Cloud CLI](https://cloud.google.com/sdk/gcloud).
- Authenticate and set project:
  ```bash
  gcloud auth login
  gcloud config set project [YOUR_PROJECT_ID]
  ```

### Build & Deploy Commands
1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Enable necessary GCP APIs:
   ```bash
   gcloud services enable run.googleapis.com containerregistry.googleapis.com
   ```
3. Submit build to Cloud Build:
   ```bash
   gcloud builds submit --tag gcr.io/[YOUR_PROJECT_ID]/vaya-backend
   ```
4. Deploy the image to Cloud Run:
   ```bash
   gcloud run deploy vaya-backend \
     --image gcr.io/[YOUR_PROJECT_ID]/vaya-backend \
     --platform managed \
     --region us-central1 \
     --allow-unauthenticated \
     --set-env-vars DATABASE_URL="postgresql://neondb_owner:password@ep-cool-snowflake-123456.ap-southeast-1.aws.neon.tech/neondb?sslmode=require" \
     --set-env-vars PORT=8080
   ```
5. Copy the HTTPS URL generated by Cloud Run (e.g. `https://vaya-backend-xyz.a.run.app`).

## 3. Web Deployment (Firebase Hosting)
Vite generates static assets that can be hosted for free on Firebase Hosting.

### Build and Package
1. Configure environment variables in `web_portal/.env` and `public_website/.env`:
   ```env
   VITE_FIREBASE_API_KEY=your-api-key
   VITE_FIREBASE_AUTH_DOMAIN=your-auth-domain
   VITE_FIREBASE_PROJECT_ID=your-project-id
   VITE_FIREBASE_STORAGE_BUCKET=your-storage-bucket
   VITE_FIREBASE_MESSAGING_SENDER_ID=your-sender-id
   VITE_FIREBASE_APP_ID=your-app-id
   ```
2. Generate distribution packages:
   ```bash
   cd web_portal && npm run build && cd ..
   cd public_website && npm run build && cd ..
   ```

### Deploy to Firebase (Multi-Site)
1. Install firebase-tools globally if not present:
   ```bash
   npm install -g firebase-tools
   ```
2. Initialize and authenticate:
   ```bash
   firebase login
   ```
3. Target hosting environments (apply targets to distinguish the sites):
   ```bash
   # Bind default site to your public consumer website
   firebase target:apply hosting public [YOUR_DEFAULT_SITE_ID]
   
   # Bind secondary custom site to your admin operations portal
   firebase target:apply hosting admin [YOUR_ADMIN_SITE_ID]
   ```
4. Deploy both sites concurrently:
   ```bash
   firebase deploy --only hosting --project [YOUR_PROJECT_ID]
   ```
5. Firebase will output your live URLs (e.g. `https://vaya.web.app` and `https://vaya-admin.web.app`).
